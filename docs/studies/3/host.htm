Study 3 Test Events

The test document loaded into the IFRAME elements fire events when a test fails or passes, or when the test document finishes with no errors. However, events don't bubble in the host document. Therefore, the host document must monitor the hosted documents for status change events and re-dispatch them properly.

Notes

1. Event handlers in the host document can not be set up before the iframe document has been loaded. Therefore, the load event on iframe.contentWindow is used to set up the test event listeners.

2. The event.detail member carries additional information on the test event, for instance, which element caused the test event.

3. The test events increment counter in the host document.
